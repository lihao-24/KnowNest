## 最新关键技术决策

- V0.3 首发接入 DeepSeek API。
- AI 调用统一走服务端 `POST /api/ai`。
- 前端不得暴露 DeepSeek API Key，不允许使用 `NEXT_PUBLIC_DEEPSEEK_API_KEY`。
- `/api/ai` 只生成候选结果，不直接覆盖知识数据。
- 用户必须预览并确认后，AI 结果才能应用到摘要、标签、分类、标题或正文。
- V0.3 需要新增 AI 使用日志和基础每日限流。
- 数据隔离继续坚持 `requireUser()` + `src/lib/db` repository 显式 `userId` 过滤，Supabase RLS 作为兜底。
