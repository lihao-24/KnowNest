## 最新关键技术决策

- V0.3 首发接入 DeepSeek API，并通过服务端 allowlist registry 支持 OpenAI-compatible Provider 扩展。
- AI 调用统一走服务端 `POST /api/ai`。
- 前端不得暴露 AI Provider API Key，不允许使用 `NEXT_PUBLIC_DEEPSEEK_API_KEY` 或其他 public key 变量承载模型密钥。
- `/api/ai` 只生成候选结果，不直接覆盖知识数据。
- `/api/ai` 必须校验 `modelId` allowlist，禁止客户端传入任意模型或 endpoint。
- 用户必须预览并确认后，AI 结果才能应用到摘要、标签、分类、标题或正文。
- V0.3 需要新增 AI 使用日志和基础每日限流。
- 数据隔离继续坚持 `requireUser()` + `src/lib/db` repository 显式 `userId` 过滤，Supabase RLS 作为兜底。
