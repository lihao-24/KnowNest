# KnowNest V0.3 AI 工作上下文摘要

## 1. 版本定位

- 项目名称：KnowNest。
- 当前版本：V0.3。
- V0.3 定位：DeepSeek 模型接入 + AI 知识整理助手 MVP。
- 核心目标：让 AI 辅助用户整理知识，但不替用户接管知识。
- 当前工作方式：主控窗口生成任务提示词，后续可切自动调度；开发过程和阶段进度记录在 `docs/operations/v0.3-workflow-status.md`。

## 2. 当前项目背景

- V0.1 已完成并部署到 Vercel。
- V0.2 已完成并上线，且已通过验证。
- V0.3 已完成并部署到 Vercel Production，生产域名 `https://knownest.vercel.app`；用户已确认线上真实使用正常。
- 当前代码栈：Next.js App Router、React 19、TypeScript、Drizzle ORM、`pg`、Supabase Auth。
- 业务认证入口：`src/lib/auth/server.ts`。
- 业务数据访问入口：`src/lib/db/*`。
- 页面和业务组件不直接访问 Supabase SDK、SQL client 或 ORM。
- 现有知识能力包括：用户认证、知识新增、编辑、删除、查看、搜索、标签、分类、列表优化、详情页优化和 Supabase RLS 权限检查。

## 4. V0.3 必做能力

- DeepSeek API 接入。
- 服务端 AI Provider 封装。
- 统一 `POST /api/ai`。
- AI 生成摘要。
- AI 推荐标签。
- AI 推荐分类。
- AI 优化标题。
- AI 整理正文。
- AI 结果预览与应用。
- AI 使用日志。
- 基础使用限制。
- Vercel 环境变量配置与线上验证。

## 5. 明确不做事项

- 不做全知识库 AI 问答。
- 不做向量数据库、embedding 检索或 RAG。
- 不做文件上传解析、图片 OCR、网页剪藏或浏览器插件。
- 不做多模型配置中心。
- 不做 AI 自动后台整理全部知识。
- 不做多人协作、知识图谱或公开分享页。

## 6. AI 接入决策

- V0.3 首发 Provider：DeepSeek。
- V0.3 已通过服务端 allowlist registry 支持 OpenAI-compatible Provider 扩展；当前 Production 已配置 DeepSeek 和 Xiaomi MiMo Token Plan 相关变量名。
- DeepSeek API 使用 OpenAI-compatible 格式。
- 服务端默认 base URL：`https://api.deepseek.com`。
- 默认模型：`deepseek-v4-flash`。
- 高质量模型预留：`deepseek-v4-pro`。
- Thinking 默认关闭。
- 模型名通过环境变量配置，不写死在业务逻辑中。
- 前端不得直接调用 DeepSeek。
- 禁止使用 `NEXT_PUBLIC_DEEPSEEK_API_KEY`。

## 7. 推荐环境变量

```env
AI_PROVIDER=deepseek
DEEPSEEK_API_KEY=
DEEPSEEK_BASE_URL=https://api.deepseek.com
AI_MODEL_FAST=deepseek-v4-flash
AI_MODEL_DEFAULT=deepseek-v4-flash
AI_MODEL_QUALITY=deepseek-v4-pro
AI_DAILY_LIMIT=20
AI_MAX_INPUT_CHARS=8000
AI_MIN_INPUT_CHARS=20
```

如启用设置页模型 allowlist，继续配置：

```env
AI_DEFAULT_MODEL_ID=deepseek-default
AI_MODEL_OPTIONS=[{"id":"deepseek-default","label":"DeepSeek 默认","provider":"openai-compatible","baseUrl":"https://api.deepseek.com","apiKeyEnv":"DEEPSEEK_API_KEY","model":"deepseek-v4-flash"},{"id":"xiaomi-mimo-token-plan-pro","label":"Xiaomi MiMo Token Plan Pro","provider":"openai-compatible","baseUrlEnv":"XIAOMI_MIMO_TOKEN_PLAN_BASE_URL","apiKeyEnv":"XIAOMI_MIMO_TOKEN_PLAN_API_KEY","model":"mimo-v2-pro"}]
XIAOMI_MIMO_TOKEN_PLAN_API_KEY=
XIAOMI_MIMO_TOKEN_PLAN_BASE_URL=https://token-plan-cn.xiaomimimo.com/v1
```

真实密钥只能放在未提交的本地环境或 Vercel 环境变量中。

## 9. 数据库方向

V0.3 需要新增 migration，但执行 migration 需要单独确认。

计划新增：

- `knowledge_items.summary`
- `knowledge_items.summary_generated_at`
- `knowledge_items.ai_updated_at`
- `ai_usage_logs`

`ai_usage_logs` 只保存元信息：

- `user_id`
- `knowledge_item_id`
- `action_type`
- `model`
- `status`
- `input_length`
- `output_length`
- `error_message`
- `created_at`

日志中不得保存完整正文。

## 10. 权限与安全约束

- 所有 AI 请求必须先 `requireUser()`。
- 对已保存知识调用 AI 时，必须用 `knowledgeItemId + userId` 校验归属。
- 用户不能通过 `knowledgeItemId` 获取他人知识内容。
- AI 结果只返回候选结果，不自动覆盖标题、正文、摘要、标签或分类。
- AI 请求失败不能修改原始知识数据。
- 每个用户每日 AI 请求默认限制为 20 次。
- 输入内容需要校验空值、过短和过长。
- DeepSeek 返回内容必须做 JSON 解析和结构校验。

## 12. 当前风险提示

- DeepSeek 和 Xiaomi MiMo Token Plan 的真实模型名、endpoint 与长期可用性需要持续以各 Provider 当前官方 API 为准。
- 真实密钥不得写入仓库、文档或对话输出；AI Provider 密钥只能放在服务端环境变量中。
- 不执行数据库 migration，除非用户明确要求并确认目标数据库。
- V0.3 已由用户确认线上可正常使用；但失败日志、每日限流达到上限、Provider 异常返回和跨用户越权等边界场景仍建议在后续 smoke / regression 中专项覆盖。
- 后续可优先补：Provider 可用性监控、AI 失败日志后台查看、限流提示体验、模型选择说明、Prompt 质量评估样例集、真实浏览器 smoke test 和统一测试聚合脚本。
