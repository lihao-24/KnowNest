## 2. V0.1 版本目标

V0.1 的目标是完成 KnowNest 的最小可用版本，让用户可以真正开始记录、整理和检索个人知识。

该版本重点不在 AI、不在复杂编辑器、不在 App 化，而在于建立一个稳定的、多设备可同步的知识库基础系统。

### 2.1 一句话目标

让用户可以在 Mac、iPhone 和其他浏览器设备上登录 KnowNest，创建、编辑、整理、搜索和同步自己的生活与工作知识条目。

### 2.2 核心目标

V0.1 需要解决以下问题：

1. 用户可以登录自己的账号。
2. 用户可以在多个设备间同步知识数据。
3. 用户可以快速创建知识条目。
4. 用户可以将临时内容先放入收集箱。
5. 用户可以通过生活 / 工作空间区分内容。
6. 用户可以通过标签、类型、状态和搜索查找内容。
7. 用户可以编辑、收藏、归档和删除内容。
8. 用户可以使用 Markdown 记录技术笔记、Prompt、链接、计划等内容。

## 3. V0.1 产品原则

### 3.1 可用优先

V0.1 的核心是尽快形成可真实使用的闭环，不追求复杂和完整。

### 3.2 记录低摩擦

新建内容时不要强制填写太多字段，用户应该可以先快速保存，再慢慢整理。

### 3.3 多设备同步优先

所有核心数据必须存储在云端，并且能在不同设备登录后保持一致。

### 3.4 结构简单

第一版只保留必要结构：空间、状态、类型、标签、收藏。

### 3.5 AI 后置

V0.1 不实现 AI 功能，但数据结构和页面设计应为后续 AI 总结、AI 标签建议、知识库问答预留空间。

## 4. V0.1 用户范围

### 4.1 当前用户

V0.1 只面向产品开发者本人使用。

当前不考虑：

- 多用户协作
- 公开分享
- 团队空间

- 商业化订阅

### 4.2 用户角色

V0.1 只有一个角色：

```text

```

用户只能访问自己创建的知识内容。

## 6. 信息架构

V0.1 采用以下信息结构：

```text
空间 Space：生活 / 工作
状态 Status：收集箱 / 已整理 / 归档
类型 Type：笔记 / 链接 / Prompt / 项目记录 / 日志 / 摘录 / 计划 / 代码片段
标签 Tags：用户自定义
收藏 Favorite：是 / 否
```

### 6.1 空间 Space

空间用于区分内容的大方向。

初始空间：

| 值 | 展示名称 | 说明 |
|---|---|---|
| life | 生活 | 健康、饮食、运动、学习、旅行、生活计划等 |
| work | 工作 | 前端开发、AI 工作流、项目经验、工具配置等 |

V0.1 不支持自定义空间。

### 6.2 状态 Status

状态用于表示内容整理阶段。

| 值 | 展示名称 | 说明 |
|---|---|---|
| inbox | 收集箱 | 刚记录、尚未整理的内容 |
| organized | 已整理 | 已经整理过的正式知识条目 |
| archived | 归档 | 不常用但需要保留的内容 |

### 6.3 类型 Type

类型用于标识内容形态。

| 值 | 展示名称 | 说明 |
|---|---|---|
| note | 笔记 | 普通知识笔记 |
| link | 链接 | 网页、文档、工具链接 |
| prompt | Prompt | AI 使用提示词 |
| project | 项目记录 | 项目方案、复盘、开发记录 |
| log | 日志 | 日常记录、阶段性记录 |
| excerpt | 摘录 | 文章、书籍、对话摘录 |
| plan | 计划 | 生活计划、学习计划、工作计划 |
| snippet | 代码片段 | 命令、代码、配置片段 |

### 6.4 标签 Tags

标签由用户自定义，用于灵活分类。

示例：

```text
React
Vue
Codex
Claude Code
ChatGPT
AI 开发
Bug 修复
性能优化
减肥
饮食
运动
日语
旅行
```

## 7. 核心用户流程

## 7.1 首次使用流程

```text
打开 KnowNest
  ↓
进入登录页
  ↓
使用账号登录
  ↓
进入首页 / 知识列表页
  ↓
看到空状态提示
  ↓
创建第一条知识
```

### 验收标准

- 未登录用户访问 App 时应被引导到登录页。
- 登录成功后进入主界面。
- 新用户没有内容时，应展示友好的空状态。

---

## 7.2 快速记录流程

```text
点击新建
  ↓
输入标题或正文
  ↓
默认进入收集箱
  ↓
保存
  ↓
知识条目出现在收集箱列表中
```

### 验收标准

- 用户可以只输入正文并保存。
- 如果用户没有输入标题，系统可以使用“未命名内容”或正文首行作为默认标题。
- 新建内容默认状态为 inbox。
- 保存后内容可在列表中看到。

---

## 7.3 整理内容流程

```text
进入收集箱
  ↓
打开某条内容
  ↓
补充标题
  ↓
选择生活 / 工作空间
  ↓
选择类型
  ↓
添加标签
  ↓
将状态改为已整理
  ↓
保存
```

### 验收标准

- 用户可以修改条目的空间、类型、状态和标签。
- 状态改为已整理后，该内容不再出现在仅筛选收集箱的列表中。
- 修改后的内容在其他设备上刷新后可见。

---

## 7.4 搜索内容流程

```text
进入知识列表
  ↓
输入关键词
  ↓
系统根据标题和正文返回结果
  ↓
用户可继续通过空间、类型、状态、标签筛选
  ↓
打开目标内容
```

### 验收标准

- 支持通过标题搜索。
- 支持通过正文搜索。
- 搜索结果应只包含当前用户自己的数据。
- 搜索和筛选可以组合使用。

---

## 7.5 收藏内容流程

```text
打开知识条目
  ↓
点击收藏
  ↓
该条目变为已收藏
  ↓
用户可以在收藏筛选中找到它
```

### 验收标准

- 收藏状态可以切换。
- 收藏内容可以通过筛选查看。

---

## 7.6 归档内容流程

```text
打开知识条目
  ↓
点击归档或修改状态为归档
  ↓
内容进入归档状态
  ↓
默认列表可以隐藏归档内容
```

### 验收标准

- 用户可以将内容设置为归档。
- 归档内容不应默认干扰主要列表。
- 用户可以通过状态筛选查看归档内容。

---

## 7.7 删除内容流程

```text
打开知识条目
  ↓
点击删除
  ↓
系统二次确认
  ↓
确认后删除
  ↓
返回列表
```

### 验收标准

- 删除前必须有确认提示。
- 删除后内容不再出现在列表中。
- 删除操作只影响当前用户的数据。


## 12. 权限与安全需求

V0.1 必须保证用户只能访问自己的数据。

### 12.1 权限原则

```text
用户只能读取自己的知识条目
用户只能创建自己的知识条目
用户只能编辑自己的知识条目
用户只能删除自己的知识条目
用户只能读取和管理自己的标签
```

### 12.2 Supabase RLS 要求

所有业务表都应该启用 Row Level Security。

需要启用 RLS 的表：

```text
profiles
knowledge_items
tags
knowledge_item_tags
```

### 12.3 验收标准

- A 用户不能看到 B 用户的数据。
- 未登录用户不能访问业务数据。
- 前端即使传入其他用户的数据 ID，也不能绕过权限。

## 13. 同步需求

### 13.1 同步方式

V0.1 采用云端数据库同步。

用户在一台设备上创建或修改内容后，另一台设备刷新页面后应能看到最新数据。

### 13.2 V0.1 不要求

V0.1 不要求实时同步。

也就是说，不要求：

```text
设备 A 修改后，设备 B 无刷新自动更新
```

只要求：

```text
设备 B 刷新或重新进入页面后可以看到最新数据
```

```text

能记录
能编辑
能整理
能搜索
能在手机和电脑上基本可用
```

只要 V0.1 能稳定完成这些能力，KnowNest 就可以开始进入真实使用阶段，并为后续 AI 辅助整理、知识库问答和 App 化打下基础。
