## 2. 设计目标

KnowNest V0.1 的页面设计目标是：

```text
用简洁、稳定、低干扰的界面，帮助用户快速记录、整理、检索和管理个人知识。
```

V0.1 不追求复杂视觉效果，不追求完整设计系统，而是优先保证：

1. 用户能快速进入记录状态。
2. 用户能清楚知道内容在哪里。
3. 用户能方便地通过空间、状态、类型和标签整理知识。
4. 用户能在桌面端和 iPhone 浏览器中基本顺畅使用。
5. 页面结构足够清晰，便于后续继续迭代 AI、PWA 和 App 化能力。

---

## 3. 页面设计原则

### 3.1 记录优先

新建入口必须明显，用户进入系统后应能快速创建内容。

### 3.2 整理延后

创建内容时不强制填写复杂信息，默认进入收集箱，后续再整理。

### 3.3 导航清晰

用户应能快速切换：

```text
全部内容
收集箱
收藏
归档
生活
工作
设置
```

### 3.4 列表高效

知识库的核心浏览方式是列表，因此列表页应重点展示：

```text
标题
摘要
标签
空间
类型
更新时间
收藏状态
```

### 3.5 表单简单

知识编辑页不做复杂块编辑器，优先使用简单稳定的 Markdown 编辑体验。

### 3.6 移动端可用

V0.1 的移动端不要求像原生 App，但必须保证：

- 不横向滚动
- 按钮可点击
- 输入框可正常输入
- 列表可正常浏览
- 编辑页可正常保存

```text

/app
```

| 路由 | 页面名称 | 说明 |

| `/app/items/[id]` | 知识详情 / 编辑页 | 查看和编辑单条知识 |
| `/app/settings` | 设置页 | 当前账号与退出登录 |

## 5. 全局布局设计

## 5.1 桌面端布局

桌面端采用经典工具型布局：

```text
┌───────────────────────────────────────────────┐
│ App Header                                    │
├───────────────┬───────────────────────────────┤
│ Sidebar       │ Main Content                  │
│               │                               │
│ Navigation    │ Page Header                   │
│ Tags          │ Search / Filters              │
│ Settings      │ List / Editor                 │
│               │                               │
└───────────────┴───────────────────────────────┘
```

### 5.1.1 左侧导航栏

建议宽度：

```text
240px - 280px
```

主要内容：

```text
产品名 KnowNest
新建按钮
全部内容
收集箱
收藏
归档
生活
工作
常用标签，V0.1 可选

```

### 5.1.2 右侧主内容区

主要内容：

```text

页面说明，可选

筛选器
内容列表或编辑器
```

## 5.2 移动端布局

移动端采用简化布局：

```text
┌──────────────────────┐
│ Top Bar              │
├──────────────────────┤
│ Page Content         │
│                      │
│ List / Editor        │
│                      │
├──────────────────────┤
│ Bottom Nav or Menu   │
└──────────────────────┘
```

### 5.2.1 顶部栏

顶部栏包含：

```text
页面标题
菜单按钮
新建按钮，可选
```

### 5.2.2 移动端导航

V0.1 可选两种方案。

#### 方案 A：底部导航

适合高频入口：

```text
全部
收集箱
新建
收藏
设置
```

优点：移动端操作快。

缺点：空间有限，不适合展示太多筛选入口。

#### 方案 B：抽屉菜单

点击菜单按钮打开侧边抽屉，展示完整导航。

优点：和桌面端结构一致。

缺点：多一步点击。

### 5.2.3 V0.1 推荐

V0.1 推荐先使用：

```text
顶部栏 + 抽屉菜单
```

原因：

- 实现简单
- 与桌面导航一致
- 不需要过早设计复杂移动底部导航


### 列表项字段

|---|---|---|
| 标题 | 是 | 没有标题时显示“未命名内容” |

| 空间 | 是 | 生活 / 工作 |

| 标签 | 否 | 最多展示前几个 |
| 收藏图标 | 是 | 表示是否收藏 |
| 更新时间 | 是 | 最近更新时间 |

## 9.6 交互行为

| 操作 | 行为 |
|---|---|
| 点击新建 | 跳转 `/app/items/new` |
| 输入关键词 | 更新搜索条件并刷新列表 |
| 选择筛选项 | 更新 URL Query 并刷新列表 |
| 点击列表项 | 跳转 `/app/items/[id]` |
| 点击收藏图标 | 切换收藏状态，V0.1 可选是否支持列表内直接切换 |
| 清除筛选 | 返回默认列表 |

---

## 9.7 空状态

当用户没有任何内容时展示：

```text
还没有知识内容
先创建第一条知识，开始搭建你的个人知识库。
[新建知识]
```

当搜索无结果时展示：

```text
没有找到匹配内容
可以尝试更换关键词，或清除筛选条件。
[清除筛选]
```

---

## 9.8 移动端表现

- 搜索框置顶。
- 筛选器可以折叠。
- 列表项使用单列卡片。
- 标签过多时换行或只展示前 2-3 个。
- 新建按钮可以放在页面顶部，也可以使用右下角悬浮按钮，V0.1 优先页面顶部。

## 13.5 表单字段设计

### 标题

- 单行输入框。
- 可以为空。
- 保存时如果为空，使用默认标题。

### 正文

- 多行 textarea。
- 支持 Markdown 输入。
- 高度应足够，不要太小。

### 空间

- select 或 radio。
- 选项：生活、工作。

### 类型

- select。
- 选项：笔记、链接、Prompt、项目记录、日志、摘录、计划、代码片段。

### 状态

- select。
- 默认收集箱。

### 标签

- 输入标签后回车添加。
- 支持删除已添加标签。
- V0.1 不强制做自动补全，但可以做简单已有标签建议。

### 来源链接

- 单行输入框。
- 可为空。

### 收藏

- checkbox 或 icon button。

---

## 13.6 保存行为

```text
点击保存
  ↓
校验标题和正文不能同时为空
  ↓
按钮 loading
  ↓
创建 knowledge_items
  ↓
创建或复用 tags
  ↓
绑定标签
  ↓
成功后跳转详情页或返回列表
```

V0.1 推荐保存成功后：

```text

```

## 13.7 错误状态

| 场景 | 提示 |
|---|---|
| 标题和正文都为空 | 请至少输入标题或正文 |
| 保存失败 | 保存失败，请稍后重试 |
| 标签为空 | 忽略空标签，不保存 |

---

## 13.8 移动端要求

- 保存按钮始终容易点击。
- 元信息区域可以折叠，避免页面太长。
- textarea 输入时不要被导航遮挡。

## 30. 页面验收标准

V0.1 页面结构完成后，应满足以下标准。

### 30.1 全局布局

- 桌面端有清晰的左侧导航和主内容区。
- 移动端可以打开导航菜单。
- 当前页面导航状态清晰。

### 30.2 登录页

- 可以输入邮箱和密码。
- 登录中有 loading 状态。
- 登录失败有错误提示。
- 登录成功跳转 `/app`。

### 30.3 列表页

- 可以展示知识列表。
- 可以搜索。
- 可以筛选。
- 可以点击进入详情。
- 空状态、加载状态、错误状态可用。

### 30.4 新建页

- 可以输入标题和正文。
- 可以设置空间、类型、状态、标签、来源链接。
- 可以保存。
- 标题和正文不能同时为空。
- 保存失败有提示。

### 30.5 编辑页

- 可以加载已有内容。
- 可以修改字段。
- 可以保存。
- 可以删除。
- 可以收藏。
- 可以归档。

### 30.6 移动端

- iPhone 浏览器中页面不横向溢出。
- 列表可正常滚动。
- 编辑器可正常输入。
- 保存按钮可正常点击。
- 导航可正常打开和关闭。

## 31. 当前结论

KnowNest V0.1 的页面结构应围绕三个核心动作设计：

```text
记录
整理
查找
```

页面不需要复杂，但必须清晰、稳定、低干扰。

V0.1 最重要的页面是：

```text
全部内容页
收集箱页
新建知识页
知识详情 / 编辑页
```

只要这些页面可用，KnowNest 就可以进入真实使用阶段。

后续版本再逐步增强：

```text
AI 辅助整理
知识库问答
PWA
iOS App
附件与导入导出
```
